📄 Dokumentasi API User

Base URL: http://202.10.35.227/api/user

1. 🔐 Login User

Endpoint: POST /login

Deskripsi: Autentikasi user berdasarkan email dan password.

Request Headers:

{
  "Content-Type": "application/json"
}

Request Body:

{
  "email": "admin@test.com",
  "password": "admin123"
}

Response Body:

{
  "message": "Login berhasil",
  "user": {
    "id": "6840e85d41891a4e96746345",
    "name": "john doe",
    "email": "admin@test.com"
  }
}
  • user: Data pengguna yang berhasil login

2. 📝 Register User

Endpoint: POST /register

Deskripsi: Mendaftarkan user baru ke sistem.

Request Headers:

{
  "Content-Type": "application/json"
}

Request Body:

{
  "name": "john doe",
  "email": "admin@test.com",
  "password": "admin123"
}

Response Body:

{
  "message": "Registrasi berhasil",
  "user": {
    "id": "6843649d4669109495ce23f6",
    "name": "john doe",
    "email": "admin@test.com"
  }
}

📄 Dokumentasi API Transaction

1. 📥 Get Transactions

Endpoint: GET /api/users/{userId}/incomes atau GET /api/users/{userId}/expenses

Deskripsi: Mengambil daftar transaksi berdasarkan jenis.

Request Headers:

{}

Response Body:

[
  {
    "nominal": 100000,
    "kategori": "Transportasi",
    "deskripsi": "Ojek coba",
    "_id": "68444fc18bdfc9598b95aeb2",
    "date": "2025-06-07T14:42:09.468Z"
  }
]

2. ➕ Add Transaction

Endpoint: POST /api/users/{userId}/incomes atau POST /api/users/{userId}/expenses

Deskripsi: Menambahkan transaksi untuk user.

Request Headers:

{
  "Content-Type": "application/json"
}

Request Body:

{
  "nominal": 100000,
  "kategori": "Transportasi",
  "deskripsi": "Ojek coba"
}

Response Body:

{
  "message": "Transaction added",
  "transaction": {
    "nominal": 100000,
    "kategori": "Transportasi",
    "deskripsi": "Ojek coba"
  }
}

3. 🗑️ Delete Transaction

Endpoint: DELETE /api/users/{userId}/incomes/{transactionId} atau DELETE /api/users/{userId}/expenses/{transactionId}

Deskripsi: Menghapus transaksi berdasarkan ID.

Request Headers:

{}

Response Body:

{
  "message": "Transaction deleted"
}

4. 🔄 Update Transaction

Endpoint: PUT /api/users/{userId}/incomes/{transactionId} atau PUT /api/users/{userId}/expenses/{transactionId}

Deskripsi: Memperbarui data transaksi berdasarkan ID.

Request Headers:

{
  "Content-Type": "application/json"
}

Request Body:

{
  "nominal": 12000
}

Response Body:

{
  "message": "Transaction updated",
  "transaction": {
    "nominal": 12000,
    "kategori": "Transportasi",
    "deskripsi": "Ojek coba",
    "_id": "68444fc18bdfc9598b95aeb2",
    "date": "2025-06-07T14:42:09.468Z"
  }
}

1. 💬 Chat API

Endpoint: POST http://202.10.35.227/chat

Deskripsi: Mengirim pesan ke chatbot untuk mendapatkan respons terkait keuangan.

Request Headers:

{
      "Content-Type": "application/json"
}

Request Body:

{
      "message": "gimana cara nabung?"
}

Response Body:

{
      "reply": "Hmm, aku masih belajar nih. Bisa coba tanya yang lain terkait tips keuangan mahasiswa?"
}
